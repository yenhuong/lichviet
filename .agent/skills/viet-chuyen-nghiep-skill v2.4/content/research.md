# Nghiên Cứu Topic → Thông Tin

**Module:** N1 — Content  
**Loại:** ⚪ Tùy chọn  
**Mục đích:** Thu thập và tổ chức thông tin từ topic khi chưa đủ dữ liệu để viết.

---

## Khi Nào Dùng

- ✅ User đưa topic nhưng thiếu thông tin chi tiết
- ✅ Chủ đề xa lạ, cần tìm hiểu
- ✅ Cần facts, context, examples để làm phong phú content
- ❌ Topic quen thuộc, đã đủ kiến thức
- ❌ User đã cung cấp đầy đủ thông tin

---

## Quy Trình

### Bước 1: Map Information Gaps

Từ topic, liệt kê 5W1H: What, Why, Who, Where, When, How

```
Topic: [Tên topic]
Đã biết: [...]
Cần tìm:
- [ ] Định nghĩa chính xác
- [ ] Statistics/numbers
- [ ] Examples cụ thể (Vietnam context)
- [ ] Expert opinions
- [ ] Trends/future outlook
```

### Bước 2: Research (Theo tier)

- **Tier 1 (Bắt buộc):** Định nghĩa, core concepts → Wikipedia, textbooks
- **Tier 2 (Quan trọng):** Trends, statistics, expert views → News, reports
- **Tier 3 (Làm phong phú):** Stories, VN examples, tips → Case studies, local media

### Bước 3: Validate

- **Credibility:** Nguồn uy tín? (.gov, .edu, reputable outlets)
- **Relevance:** Match topic? Phù hợp context VN?
- **Recency:** Còn mới? (<1 năm cho tech/trends)
- **Cross-check:** Facts quan trọng → ít nhất 2 nguồn xác nhận

### Bước 4: Organize Output

```
## Content Brief: [Topic]

### Core Message
[1-2 câu tóm tắt]

### Key Points
1. [Point + supporting info]
2. [Point + supporting info]

### Supporting Materials
- Statistics: [Số liệu + nguồn]
- Examples: [2-3 examples, ưu tiên VN]

### Vietnam Context
[Thông tin đặc thù VN]

### Content Angle
[Gợi ý approach: inspiring? practical? analytical?]
```

Sau đó chuyển sang style module phù hợp (V1/V3).

---

## Best Practices

- Start broad → go narrow
- Ưu tiên recent + local (VN)
- 3-5 sources chất lượng > 20 sources lướt qua
- Set time limit (30-60 phút) rồi bắt đầu viết
- 50-50: global best practices + VN applications
